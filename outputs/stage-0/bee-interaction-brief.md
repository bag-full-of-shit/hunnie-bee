# Idea Brief: 벌 교감 기능

## 한 줄 요약
**목표 달성으로 꿀을 모아 벌을 돌보는 교감 시스템**

---

## 배경 & 문제

### 현재 상황
Hunnie-Bee는 습관 추적 앱으로, 사용자가 목표를 등록하고 "했다" 버튼으로 달성을 기록한다. 벌 캐릭터가 있지만 단순 장식에 가깝다.

### 문제점
1. **보상 부재**: 목표 달성해도 얻는 게 없음
2. **감정적 연결 없음**: 벌과 나의 관계가 없음
3. **지속 동기 부족**: 앱을 꾸준히 켜야 할 이유가 약함

### 타겟 사용자
> "귀여운 걸 좋아하면서, 습관 형성이 어렵고, 혼자 동기부여가 안 되는 사람"

---

## 솔루션

### 핵심 컨셉
**"내가 게으르다 → 벌에게 안 좋은 영향 → 게을러지면 안 되겠다"**

- 벌과 상호 케어 관계 형성
- 귀여움(챙기고 싶음) + 미안함(열심히 해야지) 두 감정 활용
- 친구지만 돌봐줘야 하는 존재

### 핵심 루프
```
목표 "했다" → 🍯 꿀 획득 → 벌에게 꿀 주기 → 벌이 행복해짐
    ↓                                           ↑
방치하면 → 벌이 시무룩 → 죄책감 → 열심히 함 ────┘
```

## 기능 명세

### MVP (P0)

| 기능 | 설명 |
|------|------|
| **벌 탭** | 탭 네비게이션에 "내 벌" 탭 추가 |
| **벌 상태 시스템** | 행복/보통/시무룩/걱정 4단계 |
| **꿀 경제** | 목표 완료 시 꿀 +1 획득 |
| **꿀 주기** | 꿀 -1 소모, 포만감 +30 |
| **일일 감소** | 매일 포만감 -10, 애정도 -5 |

### P1 (2차)

| 기능 | 설명 |
|------|------|
| 쓰다듬기 | 1일 1회, 애정도 +20 |
| 상태별 표현 | 이모지/텍스트 변화 |
| 벌 이름 설정 | 사용자 커스터마이징 |

### P2 (향후)

| 기능 | 설명 |
|------|------|
| 이미지 에셋 | 이모지 → 실제 이미지 교체 |
| 꾸미기 | 모자, 날개 색 등 |
| 벌 성장 | 장기 보상, 진화 |

## 상태 시스템

### 수치
| 수치 | 범위 | 증가 | 일일 감소 |
|------|------|------|-----------|
| 포만감 | 0-100 | 꿀 주기 +30 | -10 |
| 애정도 | 0-100 | 쓰다듬기 +20 | -5 |

### 상태 판정
```
걱정 (Worried) : 3일 이상 교감 없음
행복 (Happy)   : 포만감 ≥ 70% AND 애정도 ≥ 70%
보통 (Normal)  : 포만감 ≥ 40% OR 애정도 ≥ 40%
시무룩 (Sad)   : 그 외
```

### 상태별 반응
| 상태 | 이모지 | 대사 예시 |
|------|--------|-----------|
| 행복 | 🐝✨ | "오늘도 같이 힘내자!" |
| 보통 | 🐝 | "오늘 하루 어때?" |
| 시무룩 | 🐝💧 | "좀 외로웠어..." |
| 걱정 | 🐝🥺 | "어디 갔어...? 걱정했어" |

## 데이터 모델

```typescript
interface BeeState {
  name: string;            // 벌 이름 (기본: "꿀순이")
  fullness: number;        // 포만감 (0-100)
  affection: number;       // 애정도 (0-100)
  honeyCount: number;      // 보유 꿀 개수
  lastFedAt: Date | null;  // 마지막 꿀 준 시간
  lastPetAt: Date | null;  // 마지막 쓰다듬은 시간
  lastInteractionAt: Date; // 마지막 교감 시간
  lastCheckedAt: Date;     // 마지막 일일 감소 체크 시간
}
```

## 기존 시스템 연동

### 변경 사항
- **goalStore**: 꿀 관련 액션 추가 (earnHoney, feedBee)
- **Goal 완료 로직**: 완료 시 `earnHoney()` 호출
- **새 저장소**: beeStore 또는 기존 store 확장

### 유지 사항
- **기존 BeeStatus**: 홈 탭에서 기존대로 동작 (진행률 기반)
- **새 벌 탭**: 별도 시스템으로 운영 (포만감/애정도 기반)

## 죄책감 밸런스 가이드

### DO ✅
- 벌이 시무룩해 보이게
- "걱정했어" 같은 따뜻한 말
- 벌집이 어두워지는 연출

### DON'T ❌
- 벌이 죽거나 떠남
- 원망하는 말 ("왜 안 와?")
- 과도한 알림 스팸

**포인트**: 원망이 아닌 걱정, 죄책감이 아닌 미안함
