# Step 0.1: Raw Idea (원본 아이디어)

## 아이디어 개요
**Hunnie-Bee 앱에 "벌 교감" 기능 추가**

## 현재 상태 분석
1. 사용자가 목표를 등록한다
2. 목표를 이루는 과정에서 할 수 있는 건 '했다' 액션뿐
3. 꿀벌 상태가 변하지만, 사용자가 얻는 가치가 없음

## 추가하고 싶은 기능
**꿀벌 교감/키우기 서비스** (새 탭)

### 핵심 컨셉
- "내가 게으르다 → 벌에게 안 좋은 영향 → 게을러지면 안 되겠다"
- 친구지만 돌봐줘야 하는 존재
- 상호 케어 관계 (벌이 나를 응원, 나도 벌을 챙김)

### 원하는 교감 액션
- 꿀 주기
- 쓰다듬기
- 목표 달성에 따른 상태 변화

### 원하는 피드백
- 외형 변화 (가장 중요)
- 약간의 죄책감 유발 (but 너무 heavy하지 않게)

## 벤치마킹 서비스
| 서비스 | 특징 | 차용할 점 |
|--------|------|-----------|
| Finch | 교감형, 벌주지 않음, 함께하는 친구 | 긍정적 상호작용, 걱정하는 느낌 |
| Habbie | 키우기형, 다마고치 스타일 | 목표→보상 연결 |
| 다마고치 | 책임감, 죄책감 | 돌봄의 책임 |

## 설계 방향

### 메커니즘
```
목표 "했다" → 꿀 획득 → 벌에게 꿀 주기/교감
방치 → 벌이 시무룩 → 죄책감 → 다시 열심히
```

### 벌 상태 시스템
- 행복: 꿀 충분 + 교감 → 날아다님, 콧노래
- 보통: 적당히 관리 → 평온한 표정
- 시무룩: 2-3일 방치 → 축 처짐
- 걱정: 오래 방치 → "어디 갔어...?" 말풍선

### 죄책감 밸런스
- ❌ 벌이 죽거나 떠남 (너무 heavy)
- ⭕ 벌이 시무룩 + 걱정하는 말
- ⭕ 벌집이 어두워짐
- 포인트: 원망이 아닌 걱정
