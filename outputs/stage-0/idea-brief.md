# Hunnie-Bee 아이디어 브리프

## 프로젝트 개요

**프로젝트명**: Hunnie-Bee
**한 줄 설명**: 1년에 100번, 목표를 꿀벌처럼 꾸준히 모아가는 습관 추적 앱
**타입**: 모바일 앱 (공개 서비스)

## 핵심 컨셉

"1년에 100번만 하면 성공"이라는 철학을 기반으로, 부담 없이 꾸준히 습관을 쌓아가도록 돕는 앱입니다. 매일 해야 한다는 압박 대신, 1년 365일 중 100번만 달성하면 된다는 접근으로 사용자에게 심리적 여유를 주면서도 지속적인 동기부여를 제공합니다.

### 네이밍 의미
- **Hunnie**: 100(Hundred)의 친근한 표현 + 꿀(Honey)
- **Bee**: 꿀벌처럼 꾸준히 모아가는 이미지

## 문제 정의

### 기존 습관 앱의 문제점
1. **매일 해야 한다는 압박감** - 하루만 빠져도 연속 기록이 깨지는 스트레스
2. **실패에 대한 죄책감** - 목표를 못 지키면 앱 자체를 포기하게 됨
3. **비현실적인 기대** - 완벽한 습관 형성을 요구

### Hunnie-Bee의 해결책
1. **100/365 철학** - 1년에 100번만 하면 성공! (약 3.6일에 1번)
2. **진행률 시각화** - 얼마나 했는지, 남은 목표가 얼마인지 직관적으로 표시
3. **긍정적 피드백** - 실패가 아닌 성취에 집중

## 핵심 기능

### MVP (Stage 3 목표)

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 목표 생성 | 연간 목표와 목표 횟수(기본 100번) 설정 | P0 |
| 진행 기록 | 버튼 한 번으로 오늘의 실천 기록 | P0 |
| 진행률 시각화 | 100번 중 몇 번 했는지 시각적으로 표시 | P0 |
| 벌집 UI | 100개의 육각형 셀이 채워지는 형태 | P0 |
| 다중 목표 | 여러 개의 연간 목표 관리 | P1 |
| 히스토리 | 언제 실천했는지 기록 확인 | P1 |

### 향후 확장 (Post-MVP)
- 리마인더/알림 기능
- 동기부여 메시지
- 통계 및 분석
- 소셜 기능 (친구와 공유)
- 클라우드 동기화 (Supabase 연동)

## 기술 스택

### 프론트엔드
- **프레임워크**: Expo (React Native 기반)
- **언어**: TypeScript
- **상태관리**: Zustand 또는 React Context
- **UI**: React Native 기본 컴포넌트 + 커스텀 스타일링

### 데이터 저장
- **1차**: AsyncStorage (로컬 저장)
- **설계 원칙**: Repository 패턴으로 추후 Supabase 등 외부 DB 연동 가능하도록 구현

### 배포
- **앱 배포**: Expo Go (개발), EAS Build (프로덕션)
- **웹 프리뷰**: Expo Web

## 타겟 사용자

### 주요 타겟
- 습관 형성에 여러 번 실패한 경험이 있는 사람
- 완벽주의 성향이지만 유연한 접근을 원하는 사람
- 장기적인 목표를 부담 없이 추적하고 싶은 사람

### 사용 시나리오
1. 새해 목표: "올해는 책 100권 읽기" → 3.6일에 1권
2. 운동 습관: "1년에 운동 100번 하기" → 주 2회 정도면 충분
3. 취미 개발: "기타 연습 100번 하기" → 부담 없이 시작

## 성공 지표

### MVP 단계
- 앱 설치 후 첫 주 내 목표 생성 비율: 80%+
- 첫 달 동안 최소 1회 이상 기록한 사용자: 60%+

### 장기 목표
- 연간 목표 달성률 (100번 완료): 30%+
- 월간 활성 사용자 유지율: 40%+

## 차별화 포인트

1. **100번 철학**: 매일이 아닌 연간 100번이라는 새로운 프레임
2. **벌집 시각화**: 100개의 육각형이 채워지는 만족감
3. **긍정적 UX**: 빠진 날이 아닌 채운 날에 집중
4. **심플함**: 최소한의 기능으로 최대의 효과
