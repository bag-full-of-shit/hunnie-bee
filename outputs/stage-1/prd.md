# Hunnie-Bee PRD (Product Requirements Document)

## 1. 개요

### 1.1 프로젝트 정보
- **프로젝트명**: Hunnie-Bee
- **버전**: 1.0.0 (MVP)
- **작성일**: 2026-02-01

### 1.2 제품 비전
"1년에 100번만 하면 성공" - 부담 없이 꾸준히 목표를 달성하도록 돕는 습관 추적 앱

### 1.3 목표
1. 사용자가 연간 목표를 설정하고 100번의 실천을 추적할 수 있게 한다
2. 벌집 형태의 시각화로 진행 상황을 직관적으로 보여준다
3. 확장 가능한 아키텍처로 향후 클라우드 동기화를 지원할 수 있도록 한다

## 2. 기능 요구사항

### 2.1 목표 관리

#### F1: 목표 생성
| 항목 | 내용 |
|------|------|
| ID | F1 |
| 우선순위 | P0 (MVP 필수) |
| 설명 | 사용자가 새로운 연간 목표를 생성할 수 있다 |

**입력 필드**:
| 필드 | 타입 | 필수 | 기본값 | 설명 |
|------|------|------|--------|------|
| title | string | Y | - | 목표 제목 (예: "책 읽기") |
| targetCount | number | N | 100 | 목표 횟수 |
| startDate | date | N | 오늘 | 시작일 |
| endDate | date | N | 시작일 + 1년 | 종료일 |
| emoji | string | N | 🐝 | 목표 아이콘 |

**비즈니스 규칙**:
- 목표 제목은 1~50자 사이여야 함
- targetCount는 1~365 사이여야 함
- endDate는 startDate보다 이후여야 함

#### F2: 목표 수정
| 항목 | 내용 |
|------|------|
| ID | F2 |
| 우선순위 | P1 |
| 설명 | 사용자가 기존 목표를 수정할 수 있다 |

**수정 가능 필드**: title, targetCount, emoji
**수정 불가 필드**: startDate, endDate (진행 중인 목표)

#### F3: 목표 삭제
| 항목 | 내용 |
|------|------|
| ID | F3 |
| 우선순위 | P1 |
| 설명 | 사용자가 목표를 삭제할 수 있다 |

**비즈니스 규칙**:
- 삭제 전 확인 다이얼로그 표시
- 삭제 시 관련 모든 기록도 함께 삭제

#### F4: 목표 목록 조회
| 항목 | 내용 |
|------|------|
| ID | F4 |
| 우선순위 | P0 (MVP 필수) |
| 설명 | 사용자의 모든 목표를 목록으로 조회할 수 있다 |

**표시 정보**:
- 목표 제목 및 이모지
- 현재 진행률 (N/100)
- 남은 일수

### 2.2 실천 기록

#### F5: 실천 기록하기
| 항목 | 내용 |
|------|------|
| ID | F5 |
| 우선순위 | P0 (MVP 필수) |
| 설명 | 버튼 한 번으로 오늘의 실천을 기록한다 |

**동작**:
1. 목표 상세 화면에서 "오늘 했어요!" 버튼 탭
2. 현재 날짜/시간으로 기록 생성
3. 진행률 즉시 업데이트
4. 축하 피드백 표시 (진동 + 애니메이션)

**비즈니스 규칙**:
- 하루에 여러 번 기록 가능 (같은 목표에 대해)
- 기록은 취소 가능 (실수 대비)

#### F6: 실천 기록 취소
| 항목 | 내용 |
|------|------|
| ID | F6 |
| 우선순위 | P1 |
| 설명 | 실수로 기록한 실천을 취소할 수 있다 |

**제한사항**:
- 24시간 이내의 기록만 취소 가능
- 취소 전 확인 다이얼로그 표시

#### F7: 실천 히스토리 조회
| 항목 | 내용 |
|------|------|
| ID | F7 |
| 우선순위 | P1 |
| 설명 | 특정 목표의 모든 실천 기록을 조회할 수 있다 |

**표시 정보**:
- 날짜별 실천 횟수
- 캘린더 뷰 (선택적)
- 총 실천 횟수

### 2.3 시각화

#### F8: 벌집 그리드 시각화
| 항목 | 내용 |
|------|------|
| ID | F8 |
| 우선순위 | P0 (MVP 필수) |
| 설명 | 100개의 육각형 셀로 진행 상황을 시각화한다 |

**레이아웃**:
- 10x10 그리드 형태의 육각형 셀
- 채워진 셀: 꿀색 (amber/gold)
- 빈 셀: 회색 (비활성)

**인터랙션**:
- 셀 탭 시 해당 기록의 날짜 표시 (툴팁)
- 진행률에 따른 애니메이션 효과

#### F9: 진행률 표시
| 항목 | 내용 |
|------|------|
| ID | F9 |
| 우선순위 | P0 (MVP 필수) |
| 설명 | 현재 진행률을 숫자와 퍼센트로 표시한다 |

**표시 형식**:
- "42/100" (현재/목표)
- "42% 달성" (퍼센트)
- "58번 남았어요" (남은 횟수)

## 4. 데이터 모델

### 4.1 Goal (목표)
```typescript
interface Goal {
  id: string;           // UUID
  title: string;        // 목표 제목
  emoji: string;        // 이모지 아이콘
  targetCount: number;  // 목표 횟수 (기본 100)
  startDate: string;    // ISO 8601 날짜
  endDate: string;      // ISO 8601 날짜
  createdAt: string;    // 생성 시각
  updatedAt: string;    // 수정 시각
}
```

### 4.2 Record (실천 기록)
```typescript
interface Record {
  id: string;           // UUID
  goalId: string;       // 연결된 목표 ID
  recordedAt: string;   // 기록 시각 (ISO 8601)
  note?: string;        // 메모 (선택)
}
```

### 4.3 AppState (앱 상태)
```typescript
interface AppState {
  goals: Goal[];
  records: Record[];
  settings: UserSettings;
  onboardingCompleted: boolean;
}
```

## 9. 리스크 및 제약사항

### 9.1 기술적 리스크
| 리스크 | 영향도 | 대응 방안 |
|--------|--------|----------|
| 벌집 UI 렌더링 성능 | 중 | SVG 대신 Canvas 사용 고려 |
| AsyncStorage 용량 제한 | 낮 | 데이터 정리 로직 구현 |

### 9.2 제약사항
- MVP는 오프라인 전용 (클라우드 동기화 없음)
- 인증/회원가입 기능 없음 (MVP)
- 단일 기기에서만 데이터 유지
